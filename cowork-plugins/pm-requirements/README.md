# PM Requirements Plugin

Requirements engineering skills for PMs — from raw discovery inputs to validated, client-ready documents.

## Commands

| Command | What it does |
|---|---|
| `/requirements-pipeline` | Full discovery pipeline from messy inputs (transcripts, Figma, rough ideas) to a Feature Requirements doc |
| `/generate-requirements` | Generate requirements from well-defined inputs (PRD, designs) — faster, one-pass |
| `/validate-requirements` | Validate a requirements doc across 15 checks (semantic + structural) |
| `/client-ready-requirements` | Transform an internal requirements doc into a VP/Director-ready client deliverable |

## Skills included

- `requirements-pipeline` — 9-stage orchestrator with Express / Standard / Full modes
- `generate-requirements` — requirements generation from any combination of inputs
- `validate-requirements` — 15-check semantic and structural validation
- `document-audit` — scan any document for stale content and contradictions
- `review-findings` — interactive walkthrough of validation or audit findings
- `update-documents` — propagate changes across multiple related documents
- `client-ready-requirements` — 11-section client-ready transformation with VP filter
