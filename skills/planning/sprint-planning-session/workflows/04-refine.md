# Workflow 04: Refine (Optional)

**Called from:** `SKILL.md` Step 7
**Next step:** None — skill completes after this
**Saves to:** `[sprint-root]/Sprint-[N]-Planning.md` (updated in place)

---

## Purpose

Apply user feedback to the Sprint Planning Session document. This workflow may loop multiple times.

---

## Step 1: Accept Feedback

The user may request changes such as:
- Reassign tickets between groups
- Change the sprint goal
- Add or remove tickets
- Adjust the "Done" checklist
- Fix factual errors
- Reword sections

---

## Step 2: Apply Changes

For each change:
1. Identify the affected section(s)
2. Apply the edit, preserving existing formatting
3. If a ticket is added or removed, update all affected sections (group tables, "Done" checklist, stretch goals)
4. If the sprint goal changes, re-validate goal-work match

---

## Step 3: Re-Run Self-Check

Run the full self-check from `workflows/03-generate.md` Step 3.

If any P1 items now fail, fix before saving.

---

## Step 4: Save Updated Document

Save the updated document to the same path: `[sprint-root]/Sprint-[N]-Planning.md`

**CHECKPOINT (Review gate):** "Updated planning doc saved. Changes: [list]. Review and confirm, or request another iteration."

**Exit condition:** User says "looks good", "approved", "confirmed", or similar.
